# Privacy & Data Safety — Поискер Android

Документ для формы **Data safety** в Google Play Console.  
Основан на реальном Django-проекте Poisker и манифесте Android-оболочки.

Сайт / политика: https://poisker.ru/privacy  
Согласие на ПДн: https://poisker.ru/consent

## Что делает Android-оболочка

Приложение показывает мобильный сайт `https://poisker.ru/` в WebView.
Оболочка **не** реализует отдельный backend и **не** хранит пароли в SharedPreferences.
Авторизация — Django session cookies через `CookieManager`.

## Permissions

| Permission | Зачем |
|------------|--------|
| `INTERNET` | Загрузка сайта |
| `ACCESS_NETWORK_STATE` | Экран offline / баннер сети |
| `CAMERA` | Только если страница запрашивает `capture` у `<input type=file>` при создании объявления |
| `POST_NOTIFICATIONS` | Показ FCM-уведомлений (Android 13+); запрос после входа, не при каждом запуске |

**Не запрашиваются:** `CALL_PHONE`, геолокация, контакты, SMS, полный доступ к хранилищу, `QUERY_ALL_PACKAGES`, Advertising ID.

## Какие данные обрабатывает сайт (Django)

По коду моделей/форм:

- **Аккаунт:** email, пароль (хэш), имя (`display_name`), телефон
- **Согласия:** условия использования, согласие на обработку ПДн (152-ФЗ), IP согласия
- **Объявления:** заголовок, описание, цена, категория, город, фото, статус модерации
- **Сообщения:** текст переписки по объявлению, опционально фото в чате (на сайте)
- **Закладки / уведомления (веб):** сохранённые объявления, in-app уведомления
- **Отзывы (веб):** оценки продавцов после сделки
- **Служебное:** просмотры, клики по телефону, жалобы, логи модерации

## Какие данные обрабатывает Android-оболочка локально

- Cookies / session сайта (системный WebView cookie store)
- Временные файлы камеры в `cache/` через FileProvider (не галерея целиком)
- Состояние WebView для восстановления после поворота
- FCM registration token + локальный `device_id` (UUID в DataStore) для регистрации устройства на backend

Оболочка **не** добавляет Firebase Analytics, рекламные SDK, crash reporter. FCM используется только для транзакционных/сервисных push.

## Регистрация и контент

- Регистрация: **да** (на сайте)
- Пользовательский контент: **да** (объявления, фото, сообщения)
- Сообщения между пользователями: **да** (на сайте)
- Геолокация устройства: **нет** (город выбирается из списка Чеченской Республики)
- Рекламная сеть / рекламный SDK в приложении: **нет** (по коду репозитория)
- Сторонние аналитические SDK в приложении: **нет** (по коду репозитория)

## Передача третьим сторонам

Данные уходят на инфраструктуру Poisker (`poisker.ru`: приложение, БД, объектное хранилище изображений, почта для писем подтверждения).
FCM-доставка идёт через Google (Firebase Cloud Messaging); на сервере Poisker хранится device token, привязанный к аккаунту.
Отдельных рекламных/аналитических третьих сторон в Android-клиенте нет.

Email-провайдер и S3/MinIO — инфраструктура backend; указывать в Data safety согласно фактическому хостингу production.

## Что отметить в Play Data safety (ориентир)

- Collected: personal info (name, email, phone), photos, other user content, messages, app activity (page views on site)
- Purpose: app functionality, account management
- Shared: нет рекламных партнёров из приложения
- Encrypted in transit: да (HTTPS)
- Users can request deletion: через сайт / поддержку (профиль и политика на poisker.ru)

Точные формулировки сверить с https://poisker.ru/privacy перед отправкой.
